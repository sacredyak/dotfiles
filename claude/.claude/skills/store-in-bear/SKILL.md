# Store in Bear

## Exclusions

**Never store these files in Bear — skip this skill entirely if the target is:**
- `CLAUDE.md`
- `DESIGN.md`

These files must be written to disk as normal files.

## Overview

Bear app is the **default storage location** for all markdown notes and PDFs. Use the `bear-notes` MCP tools — never write `.md` files to disk.

## Quick Reference

| Task | Tool |
|------|------|
| Create a new note | `bear-create-note` with title, text, and tags |
| Append to note | `bear-add-text` with note ID and text |
| Read a note | `bear-open-note` with note ID |
| Search by tag | `bear-search-notes` with tag parameter |
| Search by term | `bear-search-notes` with term parameter |
| List all tags | `bear-list-tags` |
| Add tags to note | `bear-add-tag` with note ID and tags array |

## Date Format

Every new note must include the date on the **first line** of the body, in this exact format:

```
[[D Mon/YYYY DayName]]
```

Examples:
- `[[7 Mar/2026 Saturday]]`
- `[[14 Sep/2025 Sunday]]`
- `[[10 Jan/2026 Saturday]]`

Use today's date. No leading zero on the day.

## Creating a Note

Use the `bear-create-note` MCP tool:

```
bear-create-note(
  title: "Note Title",
  text: "[[21 Mar/2026 Saturday]]\n\nNote body here...",
  tags: "1project/name,3resource/superpowers/spec"
)
```

**Rules:**
- Do NOT include `# Title` in the body — Bear adds it from `title` parameter automatically
- First line of body must be the date in `[[D Mon/YYYY DayName]]` format
- Tags are passed as comma-separated string in the `tags` parameter
- Apply the appropriate PARA tag: `1project/...`, `2area/...`, `3resource/...`, `4archive/...`
- For superpowers-generated docs, also apply the appropriate `superpowers/` tag (see below)

## Superpowers Doc Tags

Docs generated by superpowers skills use hierarchical tags under `superpowers/`:

| Doc type | Tag | Generated by |
|----------|-----|--------------|
| Design / product spec | `3resource/superpowers/spec` | superpowers:brainstorming |
| Implementation plan | `3resource/superpowers/plan` | superpowers:writing-plans |
| Brainstorm | `3resource/superpowers/brainstorm` | superpowers:brainstorming |
| Code review | `3resource/superpowers/review` | superpowers:requesting-code-review |

Always pair with a project-specific tag. Example: `#3resource/superpowers/spec #my-project`

Do NOT use flat `#superpowers` + `#design-spec` style — use the hierarchical form.

## Attaching a PDF

1. Create a note with `bear-create-note`
2. Base64-encode the PDF: `base64 -i file.pdf`
3. Use `bear-add-file` MCP tool with the note ID, filename, and base64 content

## Task Lists in Project Docs

If the note being saved contains a task list or next actions, call the `capture-to-things` skill after saving to Bear. Do not leave tasks only in Bear.

## Common Mistakes

- Writing a `.md` file to disk instead of creating a Bear note
- Putting the date anywhere other than the first line of the body
- Using wrong date format — must be `[[D Mon/YYYY DayName]]` with the `/` between month and year
- Putting tags at the top of the note body — tags always go in the `tags` parameter
- Including `# Title` heading in the body text
- Not searching for an existing note by ID before opening — always verify the note ID exists first
- Forgetting to include the date line in the note body
